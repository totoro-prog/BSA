# Интеграция веб-сервисами SOAP

## Кейс 4. SOAP-сервисы — Клуб любителей домашних животных

ИТ-специалисты города N объединились в «Клуб любителей животных» и решили создать базу данных членов клуба и их питомцев. Для ведения этой базы разработаны два веб-сервиса на основе протокола SOAP. Клиентское приложение каждый разрабатывает самостоятельно на удобном ему языке программирования либо использует SoapUI или Postman.

Первый веб-сервис отвечает за ведение (добавление, изменение, чтение, удаление) списка членов клуба, второй — за ведение списка их питомцев. Оба веб-сервиса описаны WSDL-документами в файлах member.wsdl и pet.wsdl.

### Задание 02 — Раздел `<schema>`

На основе раздела `<schema>` WSDL-документов определите:

- назначение элемента `<element>`;
- назначение атрибута `type` элемента `<element>`;
- назначение элемента `<complexType>`;
- назначение атрибута `minOccurs`;
- назначение атрибута `maxOccurs`.

Дополнительно укажите:

- что означает элемент `<sequence>`;
- что означает отсутствие атрибутов `minOccurs` или `maxOccurs`;
- что означает запись `maxOccurs="unbounded"`;
- как определить, обязателен ли элемент в XML-сообщении.

Формулировки — по одной фразе на пункт. Результат оформите в виде таблицы.

### Задание 03 — Обратное проектирование

Для Кейса 4, опираясь на имеющийся WSDL, выполните обратное проектирование — то есть восстановите требования к разработке.

Пояснение: элемент `kind` — вид животного (кот/собака/питон/хомяк).

Опишите форматы входных и выходных сообщений для операций:

- addMember;
- addPet;
- updatePet;
- getAllMembers.

Для каждого input и output укажите:

- порядковый номер;
- имя операции;
- назначение;
- тип данных (только простые типы: string/int/decimal и т.п.; сложные типы не указывать);
- признак массива (если это массив — «да», иначе оставить пустым);
- обязательность (если обязательно — «да», иначе оставить пустым);
- примечание.

Результаты оформите в виде таблицы.
