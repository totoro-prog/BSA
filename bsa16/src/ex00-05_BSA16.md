## Exercise 00 — Назначение тегов первого уровня WSDL



| Теги первого уровня | Назначение                                                                                 |
| ------------------- | ------------------------------------------------------------------------------------------ |
| Types               | определяет типы данных, используемые в сообщениях веб-сервиса (обычно с помощью схем XSD). |
| Message             | описание данных, используемых веб-сервисом                                                 |
| Port Type           | список операций, которые может выполнять сервис из раздела message                         |
| Service             | адрес, по которому доступен сервис                                                         |
| Binding             |                                                                                            |

## Exercise 02 — Раздел `<schema>`


| Элемент схемы                                                             | Назначение                                                                                         |
| ------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| элемент < element >                                                       | объявляет конкретный информационный тег в структуре XML-документа                                  |
| атрибут type элемента < element >                                         | указывает тип данных (простой или сложный) элемента                                                |
| элемент < complexType >                                                   | определение сложной структуры данных, содержащей вложенные элементы или атрибуты                   |
| атрибут minOccurs                                                         | задает минимальное количество раз, которое элемент обязан или может встретиться в данном месте XML |
| атрибут maxOccurs                                                         | задает максимальное количество раз, которое элемент может повториться в данном месте XML           |
| элемент < sequence >                                                      | указывает, что все вложенные элементы должны идти строго в том порядке, в котором они перечислены  |
| Что означает, если minOccurs или maxOccurs не указаны                     | по умолчанию их значения считаются равными "1" (элемент должен появиться ровно один раз)           |
| Что означает запись: maxOccurs="unbounded"                                | элемент может повторяться бесконечное количество раз                                               |
| Как понять, обязательно ли должен присутствовать элемент в XML-сообщении? | если атрибут minOccurs равен "1" или больше, или не указан вовсе, то элемент является обязательным |

## Exercise 04 — Обратное проектирование, дополнение

Выходное сообщение операции getAllMembers возвращает массив (список) всех членов клуба, а операции getMemberByName возвращает только одного конкретного члена клуба (или пустой результат)
