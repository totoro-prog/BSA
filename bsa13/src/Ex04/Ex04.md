# Exercise 04 — Оценка результата (по UC02)

## 1. Результаты тестирования

### Сценарий 1 — После выполнения шагов 1, 2 UC02 (проверка бронирования)

#### Сценарий 1.1

**Условие:** Система сообщает, что выбранный слот забронирован в системе клиентом. Однако при обращении к клиенту тот отрицает, что забронировал этот слот.

| Параметр  | Значение                                                     |
| :-------- | :----------------------------------------------------------- |
| Тест-кейс | TC-POS-004 (шаг 3 — проверка)                                |
| Результат | ❌ **ОТРИЦАТЕЛЬНЫЙ**                                          |
| ФР        | Система ошибочно определяет слот как забронированный клиентом. Клиент отрицает бронирование → «фантомное бронирование». |
| Дефект    | **DEF-UC02-001**                                             |

#### Сценарий 1.2

**Условие:** Система сообщает, что выбранный слот не забронирован в системе клиентом.

| Параметр  | Значение                                                     |
| :-------- | :----------------------------------------------------------- |
| Тест-кейс | TC-POS-004 (шаг 3 — проверка)                                |
| Результат | ✅ **ПОЛОЖИТЕЛЬНЫЙ**                                          |
| ФР        | Система корректно определяет отсутствие бронирования. Поведение соответствует ОР. |

### Сценарий 2 — После выполнения шагов 1–5 UC02 (создание и закрепление слота)

#### Сценарий 2.1

**Условие:** Система выполняет закрепление слота за мастером, при дальнейшей работе слот не отражается в списке свободных (незакреплённых) слотов.

| Параметр  | Значение                                                     |
| :-------- | :----------------------------------------------------------- |
| Тест-кейс | TC-POS-005 (полный сценарий)                                 |
| Результат | ✅ **ПОЛОЖИТЕЛЬНЫЙ**                                          |
| ФР        | Слот успешно закреплён за мастером. При дальнейшей работе менеджера слот не отражается в списке доступных для назначения. Слот доступен клиенту для бронирования (статус «Свободен»). |

#### Сценарий 2.2

**Условие:** Система не выполняет закрепление слота за мастером. При дальнейшей работе этот слот также можно выбрать из списка свободных.

| Параметр  | Значение                                                     |
| :-------- | :----------------------------------------------------------- |
| Тест-кейс | TC-POS-005 (полный сценарий)                                 |
| Результат | ❌ **ОТРИЦАТЕЛЬНЫЙ**                                          |
| ФР        | Система не сохранила назначение слота мастеру. Слот по-прежнему отображается как незакреплённый. Данные не сохранены. |
| Дефект    | **DEF-UC02-002**                                             |

Возможные причины: ошибка конкурентного доступа (race condition), сбой при откате предыдущей транзакции, некорректная работа кэша БД, ошибка в логике проверки (неверный SQL-запрос или условие фильтрации). Дефект воспроизводится при определённых временных условиях. Требуется анализ журнала транзакций БД и аудит-лога системы. |

## 4. Отчёт о дефекте DEF-UC02-002

| 2    | **Краткое описание**           | Система не сохраняет закрепление слота за мастером — после выполнения всех шагов UC02 слот не появляется в расписании и остаётся доступным для назначения |
| 3    | **Предусловие**                | Менеджер Смирнова О.В. (EMP007) авторизована. Мастер Соколов А.И. (EMP003) активен. Слот 24.05.2026, 15:00–16:00 не существует в системе. |
| 4    | **Шаги воспроизведения**       | 1. Менеджер открывает страницу «Ввод расписания». 2. Выбирает дату 24.05.2026, время 15:00–16:00, мастера Соколова А.И. 3. Система подтверждает: «Слот не забронирован». 4. Менеджер нажимает «Подтвердить». 5. Система сообщает: «Слот успешно создан» (или не выдаёт сообщения). 6. Менеджер открывает расписание Соколова А.И. — слот 15:00–16:00 **отсутствует**. 7. Менеджер повторно открывает страницу «Ввод расписания» — слот 24.05.2026, 15:00–16:00 **снова доступен** для назначения. |
| 5    | **Ожидаемый результат (ОР)**   | После шага 5 слот сохранён в БД со статусом «Свободен», закреплён за Соколовым А.И. Слот отображается в расписании мастера. Слот НЕ отображается в списке незакреплённых слотов. |
| 6    | **Фактический результат (ФР)** | Система не сохранила запись в БД. Слот **не появился** в расписании мастера. Слот **по-прежнему** отображается как доступный для назначения. **Отличие от ОР:** данные не были персистентно сохранены; транзакция INSERT/UPDATE не выполнена или откатилась. |
| 7    | **Постусловия**                | 1. Проверить таблицу `service_slots` на предмет созданной записи. 2. Проверить журнал ошибок приложения и БД. 3. Повторить попытку создания слота для подтверждения воспроизводимости. |
| 8    | **Окружение**                  | ОС: Windows 10 Pro 22H2, Браузер: Google Chrome 124.0, Разрешение: 1920×1080, Сервер: Ubuntu 22.04 LTS, БД: PostgreSQL 15, Бэкенд: FastAPI |
| 9    | **Приоритет**                  | **Высокий** — полностью блокирует формирование расписания    |
| 10   | **Важность**                   | **Высокая** — без закрепления слотов система не может функционировать |
| 11   | **Описание**                   | После успешного прохождения всех 5 шагов UC02 (включая положительную проверку на шаге 3) система не сохраняет слот в базе данных. Возможные причины: ошибка транзакции (COMMIT не выполнен), сбой соединения с БД после валидации, ошибка в серверной логике сохранения, конфликт блокировок (deadlock). Дефект критичен, так как полностью блокирует основной функционал менеджера — формирование расписания мастеров. Необходима приоритетная отладка серверной части: проверить логи транзакций, middleware обработки запросов, механизм коммита. |
| 12   | **Дополнительные материалы**   | — Скриншот расписания Соколова А.И. без слота. — Скриншот списка свободных слотов с повторно доступным слотом. — Серверные логи (application.log, postgresql.log). — Лог сетевых запросов (Network tab в DevTools). |
