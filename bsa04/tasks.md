# Предметная область и информационная модель

## Кейс 1. Разработка системы онлайн-записи для сети барбершопов

Руководство компании, владеющей сетью барбершопов, запланировало запуск веб-сервиса для дистанционного бронирования визитов. Данный шаг призван привлечь поток новых клиентов, разгрузить персонал и минимизировать ручную работу за счет автоматических оповещений.

Пользоваться платформой смогут как авторизованные пользователи, так и новые посетители без регистрации. В процессе оформления записи клиент выбирает категорию (парикмахерские или косметологические услуги), конкретную процедуру, мастера и подходящее свободное время. Система в автоматическом режиме отправляет напоминания через мессенджеры или СМС (Telegram, WhatsApp, VK) по графику, который настраивает администратор. После визита пользователю предлагается поставить оценку и оставить отзыв.

Со стороны персонала в системе работает менеджер (или несколько сотрудников), который формирует и корректирует расписание мастеров, координирует клиентов в ручном режиме, отмечает выполнение услуг, принимает оплату и передает финансовые данные в бухгалтерию, а также анализирует отзывы и отчетность. Сами мастера имеют доступ к системе исключительно для просмотра своего личного графика и ознакомления с отзывами о своей работе.

## Кейс 2. Создание логистической платформы для службы доставки

В период локдауна резко выросла потребность в оперативной доставке небольших партий товаров из магазинов и заведений общепита. Группа студентов решила организовать собственный логистический стартап. Суть идеи заключается в том, чтобы собирать сведения о заказах, местах сборки и адресах доставки, а затем оперативно распределять их между курьерами через единую онлайн-платформу.

На старте данные от партнеров будут собираться любыми доступными способами и заноситься в единый формат силами оператора, но для курьеров сразу разрабатывается мобильное приложение. Через него исполнитель может изучать список доступных заказов, бронировать их, забирать в точке выдачи и доставлять клиенту, мгновенно меняя статус выполнения в системе.

Для контроля за процессами предусмотрено рабочее место диспетчера, который мониторит курьеров и при необходимости переназначает заказы. Сведения о новых заказах автоматически отправляются во внешнюю бухгалтерскую программу для расчетов с поставщиками. Туда же уходит информация о выполненных доставках для расчета вознаграждения курьеров. Итоговые суммы начислений выгружаются обратно в приложение и отображаются в личном кабинете исполнителя. За регистрацию сотрудников и управление правами доступа отвечает системный администратор.

## Задание 01. Валидация модели данных с помощью CRUD-матрицы

- Разработайте проверочную CRUD-таблицу, в которой детально сопоставите:
    
    - Пользовательские роли (стейкхолдеров).

- Итоговую матрицу сохраните под именем **ex01_[префикс_продукта]_crud.xlsx**.
    

## Задание 02. Разработка словаря данных (Data Dictionary)

**Инструкция для каждого кейса:**

- Спроектируйте детальный технический справочник (словарь данных) для разрабатываемой системы.
    
- Структурируйте спецификацию, включив в неё следующие обязательные параметры для каждого элемента данных:
    
    - Бизнес-название сущности и её полей.
        
    - Уникальный технический код или мнемонику (системный идентификатор).
        
    - Развернутое смысловое описание (определение).
        
    - Физический тип данных (строка, число, дата и т.д.).
        
    - Признак обязательности заполнения поля (настройки Null / Not Null).
        
- Результаты оформите и сохраните в файл **ex02_[префикс_продукта]_dict.xxx** (расширение зависит от выбранного вами инструмента).
    

## Задание 03. Проектирование логической модели данных (ER-диаграмма)

**Инструкция для каждого кейса:**

- На основе собранных требований и словаря данных постройте логическую схему базы данных — **ER-диаграмму (Entity-Relationship diagram)**.

- Наглядно отобразите кардинальность (множественность и тип связей) между всеми объектами на схеме.

- Готовую схему экспортируйте и сохраните под именем **ex03_[префикс_продукта]_model.xxx**.
