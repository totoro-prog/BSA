## 1. Форма представления продукта (Задача: Онлайн-запись в барбершоп)

**Таблица 0.1 — Форма представления продукта**

| №    | Параметр                      | Значение                                                     |
| :--- | :---------------------------- | :----------------------------------------------------------- |
| 1    | **Устройства**                | **Клиент:** мобильный телефон (основное), планшет, ноутбук/ПК (дополнительное). **Менеджер:** ноутбук/ПК (основное), планшет (выездная работа). **Владелец (администратор):** ПК (основное). **Мастер:** мобильный телефон (основное). |

| 5    | **Специальные особенности**   | Контрастные цветовые схемы (WCAG 2.1 AA). Масштабируемые шрифты (минимум 14px). Адаптивная вёрстка (responsive design) для экранов от 320px до 1920px. Крупные зоны нажатия на мобильных устройствах (минимум 44×44px). |

## 2. Обоснование выбора

### 2.1. Обоснование выбора устройств

**Таблица 0.2 — Обоснование устройств по ролям**

| Роль         | Устройство                           | Обоснование                                                  |
| :----------- | :----------------------------------- | :----------------------------------------------------------- |
| **Клиент**   | Мобильный телефон (основное)         | Клиенты записываются на стрижку «на ходу» — в транспорте, на улице, в перерыве между делами. Смартфон всегда под рукой. Согласно аналитике, >70% бронирований в сфере услуг происходит с мобильных устройств. |
| **Клиент**   | Планшет, ноутбук/ПК (дополнительное) | Домашняя запись с ноутбука или планшета, когда клиент планирует визит заранее. Больший экран позволяет показать больше информации (расписание на неделю, фото мастеров). |
| **Менеджер** | Ноутбук/ПК (основное)                | Менеджер работает в стационарных условиях за стойкой администратора или в офисе. Управление расписанием нескольких мастеров требует большого экранного пространства (таблица с колонками: дата, время, мастер, клиент, услуга, статус). |
| **Менеджер** | Планшет (выездная работа)            | Возможность работы вне офиса (выездные мероприятия, встречи). |
| **Владелец** | ПК (основное)                        | Административные функции выполняются редко, но требуют полноценного интерфейса. Управление ролями сотрудников, просмотр логов, отчёты — требуют табличного представления. Стационарный ПК в защищённом помещении снижает риски несанкционированного доступа. |
| **Мастер**   | Мобильный телефон (основное)         | Мастер просматривает расписание между клиентами, находясь в зале или на рабочем месте. Нет необходимости в редактировании — только чтение (read-only). Смартфон не занимает место на рабочей стойке, позволяет быстро свериться со следующим клиентом. |

### 2.2. Обоснование выбора типа программного продукта

**Таблица 0.3 — Обоснование типа продукта**

| Тип                                          | Обоснование                                                  |
| :------------------------------------------- | :----------------------------------------------------------- |
| **SaaS (Software  as a Service )**           | Клиент не должен ничего устанавливать — открыл браузер и записался. Одна кодовая база работает на всех устройствах (Windows, macOS, iOS, Android). Централизованное обновление без участия пользователей. Сеть барбершопов может расширяться — новые филиалы подключаются без установки ПО. Данные хранятся на сервере, резервное копирование централизовано. |
| **Публичный интерфейс (маркетплейс-формат)** | Клиент должен легко найти услугу, мастера и свободное время. Интерфейс минималистичный, ориентированный на быстрый выбор (как в маркетплейсах). Публичная часть индексируется поисковыми системами, что привлекает новых клиентов. |
| **Корпоративная панель для персонала**       | Менеджер, мастер и владелец имеют разные права и интерфейсы. Только авторизованные сотрудники имеют доступ. Управление расписанием, справочниками, отчётами — только для персонала. |
| **Мобильный личный кабинет для мастера**     | Read-only режим — мастер не должен изменять расписание, только просматривать. Минимум кнопок, только нужная информация: клиент, время, услуга. Возможность кэширования расписания на день (при плохом интернете в зале). |

### 2.3. Обоснование выбора способов управления

**Таблица 0.4 — Обоснование способов управления**

| Способ управления                       | Устройства                            | Обоснование                                                  |
| :-------------------------------------- | :------------------------------------ | :----------------------------------------------------------- |
| **Сенсорный экран (тап, свайп, жесты)** | Мобильные устройства (клиент, мастер) | Сенсор — стандарт для смартфонов, пользователи привыкли. Свайп для прокрутки расписания, тап для выбора — интуитивно понятно. Крупные зоны нажатия (≥44×44px) для предотвращения ошибок. |
| **Мышь + клавиатура**                   | ПК / ноутбук (менеджер, владелец)     | Мышь обеспечивает точное позиционирование в таблицах с большим количеством данных. Клавиатура — быстрый ввод текста при регистрации клиентов по телефону. |
| **Навигация Tab / Enter**               | ПК / ноутбук (менеджер)               | Для менеджера — критически важно: переключение между полями без мыши ускоряет работу при телефонном обслуживании клиентов. |

### 2.4. Обоснование выбора параметров локализации

**Таблица 0.5 — Обоснование локализации**

| Параметр            | Выбранное значение | Обоснование                                                  |
| :------------------ | :----------------- | :----------------------------------------------------------- |
| **Язык**            | Русский (основной) | Сеть барбершопов работает в России, целевая аудитория — русскоязычные клиенты. |
| **Часовой пояс**    | UTC+3 (Москва)     | Все филиалы находятся в московском часовом поясе. Единый пояс упрощает управление расписанием. |
| **Формат даты**     | ДД.ММ.ГГГГ         | Российский стандарт (31.12.2026), привычный для всех пользователей. |
| **Формат времени**  | ЧЧ:ММ, 24 часа     | Исключает путаницу с AM/PM. 14:00 вместо 2:00 PM.            |
| **Формат телефона** | +7 (XXX) XXX-XX-XX | Российский формат с маской ввода, исключает ошибки при вводе. |
| **Валюта**          | Рубли (₽)          | Расчёты в рублях, символ ₽ для наглядности.                  |

### 2.5. Обоснование специальных особенностей

**Таблица 0.6 — Обоснование специальных особенностей**

| Особенность                                | Обоснование                                                  |
| :----------------------------------------- | :----------------------------------------------------------- |
| **Адаптивная вёрстка (responsive design)** | Экраны от 320px (iPhone SE) до 1920px (монитор ПК). Один сайт работает везде, не нужно разрабатывать отдельные мобильные приложения. Точки перелома: 320px, 768px, 1024px, 1920px для оптимального отображения. |
| **WCAG 2.1 AA (контрастность)**            | Клиенты с ослабленным зрением должны иметь возможность записаться. Минимальное соотношение 4.5:1 для текста, 3:1 для крупных элементов. |
| **Масштабируемые шрифты (минимум 14px)**   | 14px — минимальный размер для комфортного чтения без масштабирования. Использование rem/em для сохранения пропорций при масштабировании браузером. |
| **Крупные зоны нажатия (≥44×44px)**        | Пальцы пользователя крупнее курсора мыши. Маленькие кнопки на сенсорном экране приводят к случайным нажатиям. Стандарт Apple HIG и Material Design. |
